## 2. 7축 진단

### 축1. SEO (Search Console 기반)

**문제점:**
- 평균 순위 19.1 → 대부분 2페이지에 위치하여 클릭 발생 불가
- 23개 노출 중 대부분이 최근 4~5일에 집중 → 색인이 아직 퍼지는 중
- 노출되는 검색어가 6개 미만 → 대부분 계산기가 아직 검색결과에 미노출

**개선안:**
1. ✅ sitemap.xml에 대출이자 계산기 누락 → 추가 완료
2. ✅ robots.txt 미존재 → 생성 완료
3. ✅ 내부 링크 부재 → 전 페이지 "관련 계산기" 섹션 추가 완료
4. 각 페이지 title에 "무료", "2026" 키워드 포함 확인 → 이미 적용됨
5. 색인 오류 4건 확인 필요 (404 1건, canonical 대체 1건, 크롤링만 됨 2건)

### 축2. 콘텐츠 품질

**현재 상태:**
- 23개 계산기 모두 info-section에 "완벽 가이드" 콘텐츠 포함
- "이런 분께 유용합니다", "알아두면 좋은 정보" 등 구조화된 설명 있음
- Schema.org WebApplication 구조화 데이터 23개 모두 적용

**문제점:**
- 일부 페이지에서 "2026년 기준" 명시가 누락 (11개 페이지)
  - 누락 목록: BMI, 복리, D-Day, 출산예정일, 대출이자, 중도퇴사연차, 적금이자, 해외주식양도세, 부가세, 만나이, 칼로리
- FAQ 스키마(FAQPage) 미적용 → 검색결과에 FAQ 리치 스니펫 노출 불가
- 경쟁 사이트(calculator.co.kr 등) 대비 차별점이 명확하지 않음

**개선안:**
1. 모든 페이지 info-section에 "2026년 기준" 명시 추가 (P1)
2. FAQPage 구조화 데이터 추가로 리치 스니펫 확보 (P2)
3. 계산기별 실제 사례/예시 시나리오 추가 (P3)

### 축3. 기술 SEO

**양호한 항목:**
- ✅ HTTPS 완료
- ✅ 모든 페이지에 고유 title, meta description, OG tags 적용
- ✅ canonical URL 모든 페이지에 설정
- ✅ Schema.org WebApplication 구조화 데이터 적용
- ✅ sitemap.xml 24개 URL 포함 (대출이자 추가 후)
- ✅ robots.txt 생성 완료

**문제점:**
- ❌ GA4 (Google Analytics 4) 미설치 → 트래픽 측정 불가
- ❌ 대출이자 계산기 디자인 불일치 (색상: #1565c0/#0d47a1 vs 표준 #2563eb/#1d4ed8)
- ❌ 대출이자 계산기 폰트 불일치 (Noto Sans KR 미적용, 시스템 폰트 사용)
- ⚠️ OG image 파일(og-*.png)이 실제 존재하는지 확인 필요
- ⚠️ 페이지 로딩 속도 최적화 여지 (이미지 없이 순수 HTML/CSS/JS라 기본적으로 빠름)

**개선안:**
1. GA4 속성 생성 후 모든 페이지에 추적 코드 삽입 (P1 - 최우선)
2. 대출이자 계산기 디자인/폰트를 표준 스펙으로 통일 (P1)
3. OG image 실존 여부 확인 및 기본 이미지 생성 (P2)

### 축4. UX·전환

**양호한 항목:**
- ✅ 모바일 반응형 (viewport meta, max-width 제한)
- ✅ 숫자 입력 쉼표 포맷팅 적용
- ✅ 계산 결과 카드 UI 깔끔

**문제점:**
- ❌ 결과 공유 기능 없음 (카카오톡/링크 복사)
- ❌ 계산 후 "다시 계산" 초기화 버튼이 눈에 띄지 않음
- ❌ Enter 키 누르면 계산 실행되는지 일부 페이지에서 미확인
- ⚠️ 내부 링크 섹션 추가 완료 → 사용자 체류시간 증가 기대

**개선안:**
1. 결과 영역에 "링크 복사" 버튼 추가 (P2)
2. 카카오톡 공유 SDK 연동 (P2)
3. 키보드 Enter 이벤트 핸들러 전 페이지 확인 (P2)

## 4. 우선순위 로드맵

### Priority 1 (이번 주 - AdSense 승인 전 필수)
1. **GA4 설치** - analytics.google.com에서 속성 생성 후 전 페이지에 코드 삽입
2. **대출이자 계산기 디자인 통일** - 색상(#2563eb/#1d4ed8), 폰트(Noto Sans KR) 수정
3. **색인 오류 해결** - Search Console에서 404 페이지 확인 및 수정/삭제
4. **"2026년 기준" 누락 페이지 보완** - 11개 페이지 info-section에 최신성 표기 추가
5. **OG image 점검** - 실제 파일 존재 여부 확인, 없으면 기본 이미지 생성

### Priority 2 (2주 이내 - 유입 확장)
1. **신규 계산기 3개 추가:** 자동차세, 취득세, 환율 계산기
2. **FAQPage 구조화 데이터** - 전 페이지에 FAQ 스키마 추가 (리치 스니펫)
3. **결과 공유 기능** - 링크 복사 버튼 + 카카오톡 공유
4. **Enter 키 이벤트** - 전 페이지 키보드 입력 UX 점검

### Priority 3 (AdSense 승인 후)
1. **광고 단위 배치 전략**
   - 상단 배너 (728x90): 헤더 아래
   - 중간 네이티브 (인피드): 계산 결과와 설명 사이
   - 하단 사각형 (300x250 또는 336x280): 관련 계산기 섹션 위
2. **A/B 테스트 계획**
   - 광고 위치별 RPM 비교
   - CTA 문구 변경 테스트 ("계산하기" vs "무료 계산")
   - 관련 계산기 표시 개수 (4개 vs 6개)
3. **성과 모니터링 대시보드** - GA4 + AdSense 연동
