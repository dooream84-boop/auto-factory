# CocoFlow 콘텐츠 감사 리포트
작성일: 2026-04-18
감사 관점: **검색으로 유입된 사용자가 페이지에 도달한 직후부터 이탈까지**
감사 대상: 상위 3개 페이지 + 신규 3개 페이지 샘플링 + 전체 26개 구조 스캔

## 요약 (핵심 발견 3가지)

1. **치명: 12개 페이지에 '완벽 가이드' info-section 블록이 정확히 2회 중복 출력**
   → 구글 관점에서 동일 도메인 내 중복 콘텐츠. 색인 품질·랭킹에 직접 악영향
   → 원인 추정: 과거 자동 삽입 스크립트가 기존 블록을 제거하지 않고 append

2. **중: 상위 3개 중 전기요금 페이지에서 데이터 일관성 붕괴**
   → title/description은 "2026 최신" vs 가이드 본문은 "2024년 기준" 잔존
   → 여름 탭 UI는 있지만 JS 내부에서 일반 요금과 동일 단가 적용 (사용자 기만 성격)
   → 신뢰도 훼손 = 리턴 방문·공유·백링크 생성 불가

3. **중: 관련 계산기 내부링크가 카테고리와 무관하게 연결됨**
   → 예) 전기요금 → BMI, 만나이, D-Day (금융·생활 결제 연관 계산기 없음)
   → 사이트 회유(dwell time) 및 페이지뷰 상승 기회 누락
   → 현재 GA4 `related_calculator_click` 이벤트도 무의미한 클릭만 측정

## 3. 신규 3개 페이지 샘플링 (DSR)

자동차세/취득세도 동일 템플릿 구조로 추정.

- 스트레스 DSR 가산금리 토글 옵션 추가 (고급 모드) → 실제 은행 심사에 더 근접
- 관련 링크에 "대출이자·취득세·양도세·복리" 이미 포함됨 ✅

## 4. 전 페이지 공통 약점 (구조적)

### 4-1. E-E-A-T 신호 부재
구글 2024~2025 업데이트 이후 계산기/금융/건강(YMYL) 카테고리는 E-E-A-T 신호가 특히 중요.

현재 전 페이지에서 누락:
- **Last Updated** 가시 표시 (예: "최종 업데이트: 2026-04-18")
- **Source** 근거 링크 (한전, 국세청, 금감원, 보건복지부 등)
- **Disclaimer** 수준별 면책 고지 (YMYL 카테고리는 강한 고지 필요)
- **About** 저자·검수·연락처 (contact 링크는 있지만 페이지 내부에서 보이지 않음)

**권장**: 모든 페이지 info-section 끝에 아래 미니 박스 템플릿 일괄 삽입

```
┌─────────────────────────────────────────┐
│ 📅 최종 업데이트: 2026-04-18             │
│ 📚 근거: 근로기준법 제60조 (law.go.kr) │
│ ℹ️ 본 계산기는 참고용이며, 실제 금액·법 │
│    률 해석은 노무사·세무사와 상담하세요  │
│ ✉️ 의견: cocoflow.contact@gmail.com     │
└─────────────────────────────────────────┘
```

### 4-2. HTML 구조 오류
모든 페이지에서 `<div class="card info-section">`이 `</main>` 태그 바깥에 위치함 (electricity-calculator 기준 205줄에서 main 닫힘, 207줄부터 info-section 시작).

**영향**: 시맨틱 HTML 구조 오류. 구글 콘텐츠 주제 파싱에 약한 혼란 유발 가능.

**조치**: `</main>` 위치를 info-section 뒤로 이동 (템플릿 수정 후 sed 일괄 적용).

### 4-3. 결과 후 액션 부재
계산 결과를 본 사용자가 다음에 할 수 있는 행동이 없음:
- 카톡 공유 → 가족/동료에게 결과 보내기 (강력한 입소문 채널)
- 결과 이미지 저장 → 스크린샷 대체 (모바일 필수)
- 인쇄 → 고지서 비교용
- 즐겨찾기 → 재방문 유도

**권장**: 결과 카드 하단에 "📤 카톡 공유 | 💾 이미지 저장 | 🖨 인쇄" 3버튼 세트 추가.

### 4-4. 입력값 보존 없음
새로고침·재방문 시 입력값 초기화 → 반복 계산(예: 월마다 전기요금 확인) 불편.

**권장**: localStorage에 최근 입력값 1세트 보존 + "이전 입력값 불러오기" 버튼.

## 5. 수익 채널 다변화 준비 (Phase 2)

AdSense 심사 진행 중이지만, 승인되어도 현재 0 클릭 상태에서는 수익 0. 병행 준비가 필요함.

### 5-1. 의도 매칭 원칙
계산기는 "돈·법·건강 관련 결정을 앞둔" 사용자가 유입됨 → 해당 의도와 직결된 서비스만 전환율이 나옴.

| 계산기 카테고리 | 적합한 제휴 상품 | 수수료 가이드 |
|---|---|---|
| DSR/대출이자 | 토스·핀다·뱅크샐러드 (대출 비교) | 가입·신청 1건당 5,000~30,000원 |
| 연말정산/종합소득세/금융소득세 | 삼쩜삼·자비스 (세금 환급 서비스) | 가입 1건당 10,000~40,000원 |
| 퇴직금/연차/4대보험 | 노무114·노무법인 (유료 상담 랜딩) | CPA 5,000~15,000원 |
| 취득세/양도세/상속세 | 세무 상담 연결(카카오톡 오픈채팅) | 리드 단가 설정 가능 |
| 자동차세 | 자동차 보험 비교 (다이렉트 보험) | 가입 1건당 10,000~50,000원 |
| 출산예정일/BMI/칼로리 | 보험(실손, 태아보험) | 상담 신청 1건당 5,000~30,000원 |
| 전기요금 | 없음 (의도와 이질적) | 광고만 |

### 5-2. 준비 작업 (사전 인프라)
1. **제휴 슬롯 placeholder 삽입**: 결과 카드 바로 아래에 `<div class="affiliate-slot" data-category="...">` 빈 슬롯을 미리 넣어두면, 제휴 가입 즉시 JS로 렌더링만 하면 됨.
2. **/disclosure/ 제휴 고지 페이지**: 공정위 표시·광고법 준수. 푸터에서 링크.
3. **CTA 컴포넌트 템플릿**: 결과 후 "내 대출 한도 비교하기 (토스)" / "세금 돌려받기 (삼쩜삼)" 같은 구체 CTA 카드.
4. **제휴사 계정 리서치 보고서**: 디비디비딥·애드픽·삼쩜삼·핀다·토스 각각의 가입 요건/심사 기간/수수료 구조 문서화 (사용자가 직접 가입).

### 5-3. 트래픽 공급 (별도 Phase)
제휴 수익은 트래픽이 있어야 발생하므로 병렬로:
- 네이버 서치어드바이저 등록
- 카카오톡 공유 버튼 (결과 화면)
- 롱테일 콘텐츠 (예: "DSR 초과하면 어떻게 되나요?" 블로그형 페이지)
- Reddit/네이버 카페/오픈채팅 시딩 (1~2회)
